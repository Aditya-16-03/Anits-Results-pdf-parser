What the service now does correctly
Multi-branch aware вЂ” parses each page's own header, groups pages into 9 sections (1,692 students) by branch/semester/exam/date.
Dynamic subjects вЂ” reconstructs subject names per page from column geometry, no hardcoding in the extraction path.
Subject в‰  lab вЂ” a subject and its lab always stay distinct. Genuine theory/lab pairs (identical truncated text) are correctly paired as X / X Lab; distinct subjects that merely truncate alike (like the CN&O column) are never mislabeled as a lab.
No lost grades вЂ” duplicate subject keys are eliminated, so every grade column maps to a unique subject.
Edge cases вЂ” missing SGPA/CGPA, F and Ab (absent) grades, and incomplete rows are handled without crashing.
Production concerns вЂ” file validation, temp-file cleanup, custom exceptions with proper HTTP codes, logging, and streaming for large PDFs.
Run it with python app.py (serves on :5000) and the tests with python -m pytest -q. Subject normalization is on by default and can be turned off via the NORMALIZE_SUBJECTS=false environment variable if you ever want the raw reconstructed names.
